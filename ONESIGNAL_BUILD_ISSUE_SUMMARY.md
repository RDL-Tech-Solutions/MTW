## The Problem

The build fails with C++ linker errors:
```
ld.lld: error: undefined symbol: operator new(unsigned long)
ld.lld: error: undefined symbol: operator delete(void*)
ld.lld: error: undefined symbol: std::__ndk1::basic_string...
ld.lld: error: undefined symbol: __cxa_allocate_exception
... (hundreds more)
```

These errors indicate that the C++ standard library (`libc++`) is not being linked to the native modules (`react-native-screens`, `expo-modules-core`).

## Why This Happens

1. **Expo SDK + New Architecture**: The combination of Expo SDK with React Native's New Architecture has known build issues on Windows
2. **NDK/CMake Configuration**: The CMake configuration generated by Expo prebuild doesn't properly link the C++ STL
3. **Windows-Specific**: This issue is more common on Windows than macOS/Linux

## Solutions (In Order of Recommendation)

### Option 1: Use EAS Build (Cloud Build) ⭐ RECOMMENDED

Instead of building locally, use Expo's cloud build service:

```bash
cd app
eas build --profile development --platform android
```

**Advantages:**
- Builds in a controlled Linux environment
- No local NDK/build tool issues
- Consistent, reproducible builds
- Can download and install the APK directly

**Setup:**
1. Install EAS CLI: `npm install -g eas-cli`
2. Login: `eas login`
3. Configure: `eas build:configure`
4. Build: `eas build --profile development --platform android`

### Option 2: Disable New Architecture

Edit `app/gradle.properties`:
```properties
newArchEnabled=false
```

Then rebuild:
```bash
cd app
npx expo prebuild --clean
npx expo run:android
```

**Disadvantages:**
- Loses New Architecture benefits
- May have compatibility issues with some libraries

### Option 3: Use Expo Go for Development (OneSignal Won't Work)

For development without OneSignal:
```bash
cd app
npx expo start
```

**Disadvantages:**
- OneSignal requires native build (won't work in Expo Go)
- Can't test push notifications

### Option 4: Build on Linux/macOS

If you have access to a Linux or macOS machine:
```bash
cd app
npx expo prebuild --clean
npx expo run:android
```

The build is more likely to succeed on Unix-based systems.

### Option 5: Wait for Expo SDK Update

This is a known issue that may be fixed in future Expo SDK releases. Monitor:
- https://github.com/expo/expo/issues
- https://github.com/software-mansion/react-native-screens/issues
